materi Docker kelompok 1
  • DOCKER adalah Sebuah proyek open source yang menyediakan platform terbuka untuk developer maupun system admin untuk dapat membangun, mengemas dan menjalankan aplikasi dimanapun sebagai sebuah wadah yang ringan.
  • CARA YANG MUDAH DAN RINGAN UNTUK MEMODELKAN REALITAS Docker bergantung pada copy-on-write model sehingga mampu membuat perubahan pada aplikasi yang kita buat menjadi lebih cepat, dengan melakukan perubahan yang diperlukan
  • MEMILIKI PEMBAGIAN TUGAS KERJA YANG LOGIS Dockers dirancang untuk meningkatkan konsistensi dengan memastikan lingkungan dimana developers yang menulis kode sesuai dengan lingkungan dimana aplikasi di deployed
  • DEVELOPMENT LIFE CYCLE YANG LEBIH CEPAT DAN EFISIEN Docker ditujukan untuk mengurangi waktu siklus antara code yang sedang dibuat, code yang akan dites, dideployed, dan digunakan. Membuat aplikasi menjadi portable, mudah untuk dibuat dan dikolaborasikan
  • CONTAINER vs VIRTUAL MACHINE 1. Container dapat memungkinkan perusahaan untuk mengemas aplikasi lebih banyak ke server fisik tunggal dari mesin VM biasa 2. Container digunakan untuk menjalankan satu aplikasi sedangkan VM untuk menjalankan beberapa aplikasi 3. Container cenderung mengunci ke versi sistem operasi tertentu sedangkan VM tidak
  • ARSITEKTUR DOCKER • Arsitektur docker menggunakan client dan server (daemon). • Docker client mengirimkan request ke docker daemon untuk untuk membangun (build), mendistribusikan (pull), dan menjalankan (run) docker container. • Docker client dan server (daemon) dapat berjalan pada sistem (host) yang sama, atau dengan meremote docker client untuk menjalankan docker daemon pada host yang berbeda. • User tidak dapat langsung menggunakan docker daemon, akan tetapi untuk menggunakan docker daemon maka user menggunakan docker client sebagai perantara. • Antara docker client dan docker daemon berkomunikasi via socket menggunakan RESTful API.
